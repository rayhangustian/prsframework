About the PRS Framework

Plan–Review–Synthesize surgical planning demo

Assessing the Clinical Judgement of ChatGPT in the Planning of Complex Head and Neck Microsurgical Reconstructive Procedure

← Back to the demo

The question

The interesting question is not whether an AI can draft a surgical plan. Drafting is easy. The interesting question is whether that plan can be made safe and trustworthy enough to be useful, which requires more than a single generation step. This framework explores that question by putting a drafted plan through a structured, automated review before it is finalized, rather than presenting a first draft as if it were a final answer.

How the loop works

The pipeline mirrors a surgical team rather than a single model call. A plan is drafted, checked against explicit clinical criteria, sent back if it falls short, and only then converted into a formal note.

↻ Revision loop
Input Case
→
Operating surgeon Planner
→
4-item checklist Review Board
Manager: safety-critical vs. minor
→
Chief resident Synthesizer
→
Output Operative Note
  1. Planner (operating surgeon) drafts a structured, conditional surgical plan from the patient case, with explicit branches for clinical uncertainties rather than a single flat sequence.
  2. Surgical Review Board evaluates the plan against four criteria, oncologic soundness, reconstructive soundness, contingency planning, and clarity, then either accepts it or sends it back with concerns. A bounded number of revision rounds is allowed before the loop is forced to a decision.
  3. Manager distinguishes safety-critical rejections from minor formatting or wording objections, so the loop is strict on substance and not on style.
  4. Synthesizer (chief resident) converts the approved plan into a formal operative note, organized as a preoperative plan, an intraoperative ablation plan, an intraoperative reconstruction plan, and key contingencies.

Relation to the thesis

This demo is a companion to the undergraduate thesis named above, which assesses AI-generated surgical plans against actual surgical plans for complex head and neck microsurgical reconstruction, using independent expert raters. The demo is a way to make the mechanism behind that evaluation visible and interactive. It does not report or anticipate any thesis findings; the study is ongoing.

What is real and what is simulated

  • Live generation calls a general-purpose large language model, and the review trace shown for that run is the pipeline's actual output, not a scripted or pre-written result.
  • The bundled case library is fully synthetic and de-identified; no real patient data is used anywhere in this demo.
  • The offline demo plays a pre-prepared review sequence for a bundled synthetic case, so the demonstration works end to end without a network connection.

Scope and limitations

This is a research prototype built for academic evaluation. It is not a medical device and is not intended for clinical use. Its outputs, including the structured plans and operative notes, are illustrative rather than clinically validated recommendations.
