The question

The interesting question is not whether an AI can draft a surgical plan. Drafting is easy. The interesting question is whether that plan can be made safe and trustworthy enough to be useful, which requires more than a single generation step. This framework explores that question by putting a drafted plan through a structured, automated review before it is finalized, rather than presenting a first draft as if it were a final answer.

How the loop works

The pipeline mirrors a surgical team rather than a single model call. A plan is drafted, checked against explicit clinical criteria, sent back if it falls short, and only then converted into a formal note. Each role below is a separate AI agent with its own instructions; the excerpts shown are taken verbatim from the prompts the system runs.

Input Surgical case

A synthetic, de-identified head and neck case: diagnosis, staging, key imaging findings, and fitness for surgery.

Prompt 1 Planner operating surgeon

Drafts a case-specific plan under the protocol's strict rules: use only stated case facts, justify the primary strategy against alternatives, and declare unknowns instead of guessing.

From the actual prompt If important information is missing, do not guess. State it under "Unknowns / Clarifications Needed" and explain why it matters.

Prompt 2 Reviewer surgical board

Audits the draft for appropriateness, completeness, safety, and clinical applicability, and scores it on the same five domains and the same 1 to 5 scale used by the thesis raters. It accepts the plan or returns concerns.

From the actual prompt You are a surgical review board of senior reconstructive microsurgeons reviewing the proposed plan for this case. Your task is to audit the planner output for appropriateness, completeness, safety, and clinical applicability.
Manager. If the board rejects, a separate check decides whether the objection is safety-critical or a minor formatting note, so the loop is strict on substance and not on style.

Converts the accepted plan into a concise preoperative planning note. Formatting only: it may not add new clinical content, and it must preserve any declared uncertainty.

Relation to the thesis

This demo is a companion to the undergraduate thesis named above, which assesses AI-generated surgical plans against actual surgical plans for complex head and neck microsurgical reconstruction, using independent expert raters. The demo is a way to make the mechanism behind that evaluation visible and interactive. The demo runs the thesis protocol (Prompts 1 to 3, v2.0): the planner and synthesizer prompts are carried over nearly verbatim, and the review stage keeps the protocol's role framing and audit criteria, with its five scoring domains returned in a machine-readable form so the interface can display them. It does not report or anticipate any thesis findings; the study is ongoing.

Scope and limitations

This is a research prototype built for academic evaluation. It is not a medical device and is not intended for clinical use. Its outputs, including the structured plans and operative notes, are illustrative rather than clinically validated recommendations. Every case in the demo is synthetic; when live generation runs, the review trace shown is the pipeline's actual output.
